1. Qué es un Sistema Operativo

Un sistema operativo (SO) es el software principal que actúa como intermediario entre el hardware de una computadora y los programas del usuario. Gestiona los recursos del sistema (como CPU, memoria y almacenamiento), facilita la ejecución de aplicaciones y proporciona una interfaz para interactuar con el dispositivo.

2. Componentes de un Sistema Operativo

Los principales componentes incluyen: - Kernel (núcleo): Maneja procesos, memoria y comunicación con el hardware. - Interfaz de usuario: Permite la interacción (CLI o GUI). - Administrador de archivos: Organiza y controla el acceso a datos. - Administrador de memoria: Asigna y libera espacio en RAM. - Controladores de dispositivos: Facilitan la comunicación con periféricos.

3. Tareas del Sistema Operativo

Sus funciones clave son: - Gestionar la ejecución de procesos (asignación de CPU). - Administrar memoria RAM y almacenamiento. - Controlar dispositivos de entrada/salida (teclado, disco duro). - Garantizar seguridad y permisos de acceso. - Proporcionar servicios a aplicaciones (APIs).

4. Funciones de un Sistema Operativo

Coordina el hardware y software para: - Optimizar el uso de recursos (CPU, memoria). - Ejecutar programas de usuario. - Manejar errores y proteger el sistema. - Ofrecer una interfaz intuitiva (ej: escritorio gráfico).

5. Características de un Sistema Operativo

Destacan: - Concurrencia: Múltiples tareas aparentemente simultáneas. - Multiusuario: Varios usuarios acceden al mismo tiempo. - Seguridad: Autenticación y cifrado de datos. - Escalabilidad: Adaptación a diferentes hardware.

6. Tipos de Sistema Operativo

Clasificación principal: - Por usuarios: Monousuario (ej: Windows 10) vs. Multiusuario (ej: Linux). - Por tareas: Monotarea (obsoletos) vs. Multitarea (modernos). - Por distribución: Centralizado (un servidor) o Distribuido (ej: Kubernetes). - Especializados: Tiempo real (ej: FreeRTOS) o Móviles (Android).

7. Sistemas Operativos Para PC

Los más usados en computadoras: - Windows (Microsoft): Dominante en escritorios, con soporte amplio. - macOS (Apple): Exclusivo para Mac, conocido por su estabilidad. - Linux: Open-source (Ubuntu, Fedora), popular en servidores.

8. Sistemas Operativos Móviles

Diseñados para smartphones y tablets: - Android (Google): Basado en Linux, el más utilizado. - iOS (Apple): Exclusivo para iPhone, enfoque en seguridad. - HarmonyOS (Huawei): Alternativa reciente con integración IoT.

9. Sistema Operativo según el Ordenador y Aplicaciones

Elección por contexto: - Servidores: Linux (estabilidad) o Windows Server (facilidad). - Escritorio: Windows (compatibilidad) o macOS (diseño). - Embebidos: FreeRTOS para dispositivos pequeños (ej: IoT).

10. Compatibilidad de los Sistemas Operativos

Depende de: - Arquitectura del hardware (x86, ARM). - Controladores (drivers) disponibles. - Formatos de archivo soportados (ej: NTFS en Windows).

11. Función básica de un Sistema Operativo

Actuar como intermediario entre el usuario y el hardware, permitiendo ejecutar software y administrar eficientemente los recursos del sistema (CPU, memoria, dispositivos).

12. Tres Sistemas Operativos más utilizados

Según cuota de mercado (2023): 1. Windows (PCs). 2. Android (móviles). 3. macOS/iOS (Apple).

13. Tareas que realizan los Sistemas Operativos

Ejemplos concretos: - Asignar memoria a un programa. - Gestionar la impresión de un documento. - Proteger datos con contraseñas. - Ejecutar actualizaciones en segundo plano.

14. Sistema operativo multitarea

Permite ejecutar múltiples aplicaciones "simultáneamente" mediante técnicas como: - Time-sharing: Alterna rápidamente entre procesos. - Multiprocesamiento: Usa varios núcleos de CPU. Ejemplo: Windows 10 al abrir navegador y editor de texto a la vez.

15. Sistema operativo multiusuario

Facilita que varios usuarios accedan al mismo equipo con: - Cuentas independientes. - Permisos personalizados. - Aislamiento de datos. Usado en servidores (Linux) o empresas (Windows Server).

16. Cómo funciona un sistema operativo

Flujo básico: 1. Al iniciar, el BIOS/UEFI carga el kernel. 2. El kernel gestiona hardware y lanza servicios esenciales. 3. Carga la interfaz de usuario (GUI o CLI). 4. Ejecuta programas asignando recursos bajo demanda.

17. Uso en entorno gráfico

Interacción mediante: - Ventanas, iconos y menús desplegables. - Acciones con ratón/táctil (arrastrar, hacer clic). - Escritorio personalizable (ej: widgets en Windows/macOS). Ejemplos: Explorador de archivos o Panel de Control.

18. Inicio del funcionamiento del SO

Proceso de arranque (boot): 1. POST (autochequeo del hardware). 2. BIOS/UEFI busca el bootloader. 3. Carga el kernel en memoria RAM. 4. Inicia servicios y muestra la pantalla de login.

19. Interfaces del Sistema Operativo

Dos tipos principales: - CLI (Línea de comandos): Comandos de texto (ej: Terminal en Linux). - GUI (Interfaz gráfica): Elementos visuales (ej: Escritorio de Windows).

20. Proceso en sistemas operativos

Un programa en ejecución con: - Su propio espacio en memoria. - Estado (ejecutando, en espera). - Identificador único (PID). El SO los gestiona para evitar conflictos.

21. Sistema distribuido y centralizado

Diferencias clave: - Centralizado: Un servidor principal controla todo (ej: mainframes). - Distribuido: Recursos compartidos en red (ej: clusters con Kubernetes).

22. Sistemas de archivos

Estructuras para organizar datos: - NTFS (Windows): Soporta permisos avanzados. - APFS (macOS): Optimizado para SSDs. - ext4 (Linux): Alto rendimiento. - FAT32: Compatible universal (USB).

23. Otras características actuales

Tendencias recientes: - Virtualización (ej: Hyper-V, Docker). - Integración con la nube (OneDrive, iCloud). - Soporte para IA (asistentes como Cortana). - Seguridad mejorada (Windows Hello).

24. Controladores en un SO

Permiten al sistema operativo comunicarse con hardware específico: - Traducen instrucciones del SO al dispositivo. - Ejemplos: Drivers para tarjetas gráficas NVIDIA o impresoras HP. - Sin ellos, el hardware no funcionaría correctamente.
